
### Development Best Practices

1. **Keep Browser DevTools Closed** - Open DevTools only when needed as they slow down rendering

2. **Use Fast Refresh** - Hot Module Reloading should work automatically:
   - Edit a file and save
   - Browser updates instantly without full page reload

3. **Monitor Memory Usage** - If dev server slows down:
   ```bash
   # Restart dev server
   # Kill any stale node processes
   ps aux | grep node
   ```

4. **Lazy Load Heavy Modules** - For components using:
   - Maps (Leaflet)
   - Data tables
   - Charts
   Use dynamic imports:
   ```typescript
   import dynamic from 'next/dynamic';
   const MapComponent = dynamic(() => import('./Map'), { ssr: false });
   ```

5. **Optimize CSS-in-JS** - If using styled-components:
   - Keep styled component definitions outside render functions
   - Use CSS Modules when possible

6. **Bundle Analysis** - Profile bundle size:
   ```bash
   npm run build -- --analyze
   # or check .next/static/chunks
   ```

## Further Optimization Options

If still experiencing slowness:

1. **Disable Type Checking in Dev**
   - Edit `tsconfig.json`: Set `skipLibCheck: true`
   - Or use `--no-type-check` flag

2. **Limit File Watching**
   - Edit `next.config.js`: Configure watchPathIgnorePatterns
   - Exclude node_modules, .git, etc.

3. **Use SSD Storage**
   - Ensure project is on SSD, not HDD

4. **Increase Node Heap Size**
   ```bash
   NODE_OPTIONS=--max_old_space_size=4096 npm run dev
   ```

5. **Profile with Chrome DevTools**
   - Use `npm run dev:profile` and open chrome://inspect

## Troubleshooting

**Dev server won't start:**
```bash
lsof -i :3000
kill -9 <PID>
npm run dev
```

**High memory usage:**
- Restart dev server
- Check for memory leaks in components
- Monitor browser for leaked event listeners

**Slow HMR:**
- Check for large files being modified
- Verify no infinite loops in useEffect
- Check for heavy computations in render

**Port already in use:**
```bash
npm run dev:turbo -- -p 3001
```
